## Decisions

- Keep-alive preference: chose **global `crush.json` via `options.keep_alive_enabled`** over `localStorage` (no cross-browser sync, invisible) and over per-project `.crush/crush.json` (would coerce coworkers when committed). Default ON via `*bool` nil-means-true; `buildConfigWire` always sends explicit bool so the frontend never sees `undefined`.
- GLM-5.2 effort: chose **two-tier selector (`high` / `max`)** to mirror docs.z.ai/devpack/latest-model's official mapping table; lower tiers from a generic 5-tier picker collapse to `high`. Detection by model id (`^glm-5...`), not by provider id, so custom-named providers (`z-ai`, `zhipu`) still get the picker.
- Stream watchdog: chose **sequential retries** (`stream_stall_retries: 4` + `stream_idle_timeout_seconds: 180`) over hedged-requests because hedging would 3× input-token cost and accelerate z.ai's 5-hour quota wall. Sequential retries already existed in `coordinator.go:476–514` — no code, two JSON keys.
- Sitter: chose **localStorage** for its persistence (rather than backend) — purely client-side behaviour, no need for backend round-trip on every toggle. Resume prompt is the single Russian word "продолжай"; agent picks up the language from history context.
- npx claude variants: chose **delete** rather than refactor — Windows ConPTY relay through `npx.cmd` was always flaky, and the variants doubled the model list for no benefit. Atom registry still has `opus`/`opus48`/etc. alias entries.
