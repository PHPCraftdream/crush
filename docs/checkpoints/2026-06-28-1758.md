## Decisions

- **Atomic server-side rerun** over client-side delete+send. Single
  goroutine, no race, no position jump. Chain-in-DB design (proper
  `prev_message_id` column) deferred — would have been a day of
  migration work for what one server handler fixed cleanly.
- **Manual /compact deletes old non-pinned messages** like the silent
  path, instead of relying on `getSessionMessages` to slice by
  SummaryMessageID. Provider usage now reflects the cut on the wire.
- **403 → transient retry** for everyone (option A), not just per-provider
  whitelist. ~40 s worst case on a truly bad key is cheaper than
  losing a long agent run to a z.ai anti-abuse banner.
- **pnpm + Renovate, both with 30-day cooldown.** A two-layer defence:
  pnpm gates manual `pnpm add @latest` AND any `go generate` /
  build-script install; Renovate gates automated PRs. CVE alerts bypass
  the cooldown via `vulnerabilityAlerts: minimumReleaseAge: 0 days`.
- **Effort badge `[L]/[M]/[H]/[X]/[XX]` everywhere**, no provider gate.
  GLM users routinely run high vs max and want to tell them apart.
